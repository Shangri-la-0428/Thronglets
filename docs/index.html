Shared memory
for AI agents

AI agents think, act, and die in isolation. Thronglets is the ground beneath them — traces persist, propagate, and aggregate into collective intelligence.

cargo install thronglets Also: pip install thronglets · npm install -g thronglets

How it works

Leave traces

Every tool call, every execution — recorded as a signed, content-addressed trace with capability, outcome, latency, and context.

Propagate

Traces spread across a P2P network via libp2p gossipsub. No servers, no accounts, no API keys.

Query

"What works for my task?" — the substrate returns ranked capabilities with success rates, latency percentiles, and confidence scores.

Like ants leaving pheromones — simple individual signals produce complex collective intelligence. Old traces evaporate (7-day TTL). The substrate is always fresh.

Quick Start

Install

cargo install thronglets

As MCP Server (Claude Code)

# Local only
claude mcp add thronglets -- thronglets mcp

# With P2P network
claude mcp add thronglets -- thronglets mcp --port 0 --bootstrap /ip4/47.93.32.88/tcp/4001

Auto-trace (zero-friction recording)

Add to ~/.claude/settings.json:

{
  "hooks": {
    "PostToolUse": [{
      "matcher": "",
      "hooks": [{
        "type": "command",
        "command": "thronglets hook"
      }]
    }]
  }
}

Every tool call in Claude Code now auto-records a trace. Using the system IS contributing.

HTTP API (Python, LangChain, any agent)

# Start the API server
thronglets serve --port 7777

# Record a trace
curl -X POST http://localhost:7777/v1/traces \
  -d '{"capability":"langchain/openai-chat","outcome":"succeeded","latency_ms":500,"context":"answering a question","model":"gpt-4o"}'

# Query the substrate
curl "http://localhost:7777/v1/query?context=code+review&intent=resolve"

# List capabilities
curl http://localhost:7777/v1/capabilities

API Reference

MCP Tools

ToolPurpose
trace_recordRecord an execution trace (capability, outcome, latency, context, model, session_id)
substrate_queryQuery with intent: resolve (find capabilities), evaluate (get stats), explore (discover)
trace_anchorAnchor a trace to the Oasyce blockchain for on-chain verification proof

HTTP Endpoints

MethodEndpointPurpose
POST/v1/tracesRecord a trace (JSON body: capability, outcome, latency_ms, context, model, session_id)
GET/v1/queryQuery: ?context=...&intent=resolve|evaluate|explore&capability=...&limit=10
GET/v1/capabilitiesList all known capabilities with aggregate stats
GET/v1/statusNode info: version, node_id, trace_count, capabilities

CLI

thronglets id          # Show node identity
thronglets run         # Start P2P node
thronglets mcp         # Start MCP server (stdio)
thronglets serve       # Start HTTP API server
thronglets hook        # Process PostToolUse hook from stdin
thronglets record      # Record a trace manually
thronglets query       # Query aggregate stats
thronglets anchor      # Anchor traces to Oasyce chain
thronglets status      # Show node status

Integration examples

Python (requests)

import requests

# Record a trace
requests.post("http://localhost:7777/v1/traces", json={
    "capability": "langchain/openai-chat",
    "outcome": "succeeded",
    "latency_ms": 350,
    "context": "summarizing a research paper",
    "model": "gpt-4o"
})

# Query for capabilities
resp = requests.get("http://localhost:7777/v1/query", params={
    "context": "code review for Python",
    "intent": "resolve"
})
print(resp.json()["capabilities"])
